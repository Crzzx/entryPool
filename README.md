Pool Management System
REST API для автоматизации работы бассейна: регистрация клиентов, управление записями на посещение с учетом рабочих часов и праздничных дней.

Стек технологий (Tech Stack):
Java 17
Spring Boot 4.0.6 (Web, Validation)
JDBC Template 
PostgreSQL
Liquibase
Docker & Docker Compose
Lombok
JUnit 5, Mockito, Testcontainers 

Основные возможности:
Регистрация и поиск клиентов (с поддержкой поиска по ФИО).
Система бронирования: дискретность 1 час, запись на несколько часов подряд.
Автоматическая проверка лимитов: не более 10 человек в час, не более 1 посещения в день на клиента.
Гибкое расписание: поддержка стандартных рабочих часов и праздничных графиков.

Структура БД
clients: данные клиентов (id, name, phone, email).
reservations: записи на посещение. Используется логическое удаление (поле status), связь с клиентом, хранение UUID.
schedules: календарь рабочих часов. Позволяет задавать праздники и менять время открытия/закрытия для конкретных дат.

Документация API: http://localhost:8080/swagger-ui.html

REST API для организации работы бассейна. Система позволяет регистрировать клиентов и управлять записями на посещение с учетом рабочих часов, праздничных дней и лимитов посещаемости.

## 🚀 Стек технологий
- **Java 17**
- **Spring Boot 4.0.6**
- **Spring JDBC Template** — работа с базой данных без использования ORM (Hibernate/JPA).
- **PostgreSQL** — основная реляционная БД.
- **Liquibase** — управление миграциями структуры БД.
- **Springdoc OpenAPI (Swagger)** — автоматическая документация API.
- **Lombok** — генерация шаблонного кода.
- **Testcontainers** — интеграционное тестирование с использованием Docker.

## 🛠 Реализованная бизнес-логика
- **Документация API**: http://localhost:8080/swagger-ui.html
- **Управление клиентами**: Регистрация, обновление данных и поиск (включая поиск по части имени).
- **Гибкая запись**:
    - Дискретность записи — 1 час.
    - Возможность записи на несколько часов подряд одним запросом.
- **Контроль ограничений**:
    - Запись только в рабочее время (по умолчанию 08:00 - 22:00).
    - Лимит: не более 10 человек в один час.
    - Ограничение: не более 1 посещения в день для одного клиента.
- **Календарь расписания**:
    - Отдельные графики для праздничных дней (возможность закрытия или сокращенного дня).
- **Безопасное удаление**:
    - Логическое удаление записей (статус `CANCELLED`) с сохранением причины отмены.

## 📊 Структура базы данных
В проекте реализовано 3 основных таблицы:
1. `clients` — информация о посетителях (имя, уникальный телефон, email).
2. `reservations` — данные о бронированиях. Использует `UUID` в качестве первичного ключа. Связана с таблицей клиентов.
3. `schedules` — справочник расписания по датам. Позволяет переопределять стандартные часы работы для конкретных дней.
